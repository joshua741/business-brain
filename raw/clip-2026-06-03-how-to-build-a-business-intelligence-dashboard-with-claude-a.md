# How To Build a Business Intelligence Dashboard With Claude AI
**Source:** https://www.youtube.com/watch?v=r2RsgtQ_z2s   **Duration:** 00:35:45

## Summary
This video provides a training session on leveraging AI, specifically Claude Code, to build a custom business dashboard without requiring extensive technical or coding knowledge. The speaker, who identifies as having no tech background, aims to demonstrate how individuals can create an application that acts as a "source of truth" for their business metrics, enabling better decision-making and daily focus. The app is designed to communicate the most important KPIs, such as lead generation, conversion rates, and disposition metrics, by integrating with existing business tools like a CRM (GoHighLevel) and a database (Supabase).

The core thrust of the training is to empower business owners to create tailored applications that address their specific operational needs, rather than relying on generic, potentially ineffective, off-the-shelf solutions. The speaker introduces concepts like "scaffolding" for organizing app features and "MCP servers" for inter-application communication, simplifying complex development processes. By walking through the step-by-step creation of a real estate business dashboard, the video illustrates how AI can act as a co-pilot, guiding users through the development process and making advanced app building accessible to a broader audience.

## Key Takeaways
*   **AI-Powered App Development is Accessible**: Users can build custom business applications with AI tools like Claude Code even without a traditional tech or coding background.
*   **"Source of Truth" for Business Metrics**: The primary goal is to create an app that aggregates and presents key business metrics, enabling informed decision-making and prioritization.
*   **Scaffolding for Structure**: "Scaffolding" is a crucial initial step in app development, involving the systematic organization of features and their underlying components.
*   **Importance of Planning**: Thorough planning of desired features and functionalities before coding helps avoid issues and ensures a more efficient development process.
*   **Inter-Application Communication (MCP)**: An "MCP server" (Model Context Protocol) acts as a universal translator, allowing different applications and AI models to communicate and exchange data seamlessly, similar to Bluetooth for devices.
*   **Leveraging Existing Tools**: The demonstration integrates with Supabase (for database management) and GoHighLevel (as a CRM) to pull and display real-time business data.
*   **KPI-Driven Dashboard Design**: The custom dashboard tracks specific KPIs across different business phases (e.g., lead generation, lead conversion, disposition) with visual indicators (red/green) for performance.
*   **Automation for Data Updates**: Implementing cron jobs can automate the syncing of data from integrated platforms to the dashboard, ensuring up-to-date metrics.
*   **Future Focus on "Skills"**: The next training session will delve into building "skills" for AI agents, enhancing their capabilities beyond basic data processing.

## Frameworks / SOPs
*   **Scaffolding**: This is a process used in app development to organize all the different features and their components. It involves thinking through the necessary functionalities (e.g., calculators, selectable comparables, visual indicators) and structuring the project accordingly. The AI assistant helps in this initial organization by creating folders and files based on the user's requirements.
*   **Model Context Protocol (MCP) Server**: Described as a "Bluetooth for all applications," an MCP server facilitates communication between different platforms or applications, especially when they don't have direct API integrations. It allows one platform to output data in a format that another platform can understand, ensuring seamless data exchange and interoperability between AI agents and various apps.
*   **Three-Phase Business Process**: The speaker outlines a specific business process divided into three phases, each with its own Key Performance Indicator (KPI) metrics:
    1.  **Phase 1: Lead Generation**: Focuses on outbound messaging (text/email to agents/gatekeepers) and tracks metrics like message deliverability rate, response rate, and opt-out rate.
    2.  **Phase 2: Lead Conversion**: Involves converting leads into deals under contract. It tracks metrics related to calls (e.g., deals stuck in "second call" longer than 48 hours), underwriting (comping properties), and closing offers (offers sent, offers signed).
    3.  **Phase 3: Disposition**: Aims to build a quality buyers list and efficiently move contracted deals to buyers. It tracks "time to blast" (how long it takes to get a deal to the buyers list) and the closing rate of deals once they hit the disposition pipeline.

## Action Items
*   **Understand AI Fundamentals**: Focus on understanding the core concepts behind AI technology to effectively leverage it, rather than just learning to code.
*   **Plan App Features (Scaffolding)**: Before starting development, clearly define all desired features and functionalities for your custom app.
*   **Set Up Supabase**: Create a Supabase account (free tier is available) and run the provided SQL schema to set up the necessary database tables for your dashboard.
*   **Obtain API Keys**:
    *   Generate a private integration token and grab your location ID from your GoHighLevel account.
    *   Obtain an API key from platform.cloud.com (Anthropic) for AI briefings.
*   **Configure Environment Variables**: Create an `.env.local` file in your project folder and paste in all the required API keys and passwords (Supabase URL, Anon/Public Key, Service Role/Secret Key, GoHighLevel Private Token, GoHighLevel Location ID, Anthropic API Key, Dashboard Password, Auth Secret, CRM Secret, Next Public App URL).
*   **Start the Application Locally**: Open your terminal in the project folder and run `npm run dev` to start the app and view it in your browser (e.g., `http://localhost:3000`).
*   **Iterate on Design and Functionality**: Review the locally running app, make design tweaks (e.g., CSS adjustments, color changes), and add/modify features as needed.
*   **Automate Data Sync**: Implement a cron job to automatically update the dashboard with fresh data from your CRM (e.g., every six hours).
*   **Deploy the Application**: Once satisfied with the local build, deploy the app to a platform like Vercel (linking it with a GitHub repository) to make it live and accessible.
*   **Explore AI Skills (Next Steps)**: Attend future training sessions to learn how to build "skills" and "skill packs" for your AI agent, enhancing its capabilities and making it more useful.
